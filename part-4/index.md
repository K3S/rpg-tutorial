# Part 4: Going Further

Parts 1–3 covered the core. You can connect to an IBM i, write modern RPG, build real programs, and test them. That's the foundation.

Part 4 is different: **it's a map, not a tutorial.**

Once you've got the fundamentals, you don't need another tutorial. You need a curated map of where to go next — the places the RPG community has already built excellent material that would be wasteful for this site to try to replace.

This part is a work in progress. As the site matures, it'll grow into a directory of trusted external resources:

- **Scott Klement's writing** — decades of deep technical RPG articles
- **IBM's RPG Programmer's Guide** — the authoritative language reference
- **Code for IBM i documentation** — the extension's own docs, including advanced workflows
- **PUB400 community** — resources and conversations around Holger Scherer's free server
- **IBM i conferences and communities** — COMMON, IBM TechXchange, i-UG, MIDRANGE-L
- **Blogs and newsletters** — Nick Litten, IBM i Magazine, Rob Berendt, Jon Paris & Susan Gantner
- **Topics this tutorial intentionally didn't cover** — display files and subfiles, Integrated Web Services, service program versioning, RDi-specific workflows, Node.js integration, XMLSERVICE for PHP/Python interop

If you want to help build this out, open a PR or an issue on the [GitHub repo](https://github.com/K3S/rpg-tutorial) — community contributions are exactly how Part 4 should grow.
